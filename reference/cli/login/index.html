swarm login: Login with the CLI

The swarm login command provides a means to authenticate a Giant Swarm account using the CLI.

The command can be used in interactive mode or credentials can be passed via the command line in non-interactive mode.

Command Syntax

Interactive Login

An interactive login session can be started by doing the following:

swarm login

Alternately, a username can be used by substituting the <username> for the desired account’s username:

swarm login <username>

Example with Response

$ swarm login
Username or email: bant
Password:
Login Succeeded
Environment bant/dev has been selected

Note: The last line of output shos which environment has been selected. The default is <yourusername>/dev which can be changed using the swarm env command.

Non-Interactive Login

In non-interactive mode, the username and password can be passed to the login command as arguments:

swarm login <username> -p <password>

Example with Response

$ swarm login bant -p f00bar
Login Succeeded
Environment bant/dev has been selected

Authentication Tokens

When logging in using the swarm login command, authentication tokens are created and stored locally, by default, in ~/.swarm/token:

$ ls -la ~/.swarm/
total 24
drwxr-xr-x   5 bant  staff   170 May 18 23:40 .
drwxr-xr-x+ 53 bant  staff  1802 May 28 11:11 ..
-rw-------   1 bant  staff    34 May 18 23:40 config
-rw-------   1 bant  staff    39 May 27 11:24 last_update_info
-rw-------   1 bant  staff    36 May 27 20:30 token

Giant Swarm authentication tokens are good for up to 30 days. To minimize the risk of unauthorized use, please use the swarm logout command to explicitly expire the authenticated token and session.

The API documentation contains more information about acquiring and using authentication tokens.
