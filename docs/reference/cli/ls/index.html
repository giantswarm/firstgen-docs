swarm ls: List Services

The swarm ls command shows a list of services and their status for the current selected environment.

Command Syntax

The command swarm ls is used without parameters to view a list of running services in a given environment:

swarm ls

The command may also be used with the -a flag to view a list of running services across all environments:

swarm ls -a

Example with Response

$ swarm ls -a
2 services available in all environments:

service          environment  created              status
helloworld       bant/dev     2015-05-09 11:07:45  up
tothemoon        bant/prod    2015-05-18 06:43:11  down

Response Explained

  • service - The service name as defined upon creation.
  • environment - The name of the environment in which the service is running.
  • created - The service creation datetime in UTC.
  • status - The status of the service.

Status Explained

The service status is an aggregate status of of all component instances within a service. Status may be one of the following:

  • up - All component instances are running.
  • starting - One or more of the component instances are in starting state.
  • stopping - One or more of the component instances are in stopping state.
  • down - One or more of the component instances are currently down.
  • failed - At least one of the components instance are currently in a failed state.
