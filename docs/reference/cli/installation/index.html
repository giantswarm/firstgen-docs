Installing the Giant Swarm CLI

Instructions for installing the swarm command line interface on the supported platforms. By the way, the latest version is 0.27.0.

Mac OS X

Mac OS X install via Homebrew

If you’re on a Mac, the recommended way to install the swarm command line interface (CLI) is to use Homebrew.

Once Homebrew is set up, these commands will install the latest swarm CLI in your PATH.

$ brew tap giantswarm/giantswarm
$ brew install swarm-client

In order to update the swarm CLI to the latest version, use these commands:

$ brew update
$ brew upgrade swarm-client

Mac OS X manual install

If you don’t want to use Homebrew, here is how you could download and install the CLI manually.

$ curl -O https://downloads.giantswarm.io/swarm/clients/0.27.0/swarm-0.27.0-darwin-amd64.tar.gz
$ tar xzf swarm-0.27.0-darwin-amd64.tar.gz

To check the integrity of the downloaded .tar.gz file, we provide SHA1 and MD5 checksum files.

Please add the swarm binary to your PATH.

Linux

Please note that the swarm CLI requires a 64 bit system.

First, download the tarball and unpack it:

$ curl -O https://downloads.giantswarm.io/swarm/clients/0.27.0/swarm-0.27.0-linux-amd64.tar.gz
$ tar xzf swarm-0.27.0-linux-amd64.tar.gz

To check the integrity of the downloaded .tar.gz file, we provide SHA1 and MD5 checksum files.

We recommend to make the swarm binary available in your PATH by copying it to a directory that’s already contained in your PATH. For example:

$ sudo cp swarm /usr/local/bin/

Windows

Currently, we don’t offer the swarm CLI for Windows. However, you can run it in a Linux virtual machine like the one used by Boot2Docker. Note that this, like Docker itself, requires a 64 bit version of Windows. Follow this procedure for installing the CLI:

  1. Start Boot2Docker
  2. Log in to Boot2Docker’s virtual machine via the command line using the command boot2docker ssh
  3. Execute curl -O https://downloads.giantswarm.io/swarm/clients/0.27.0/swarm-0.27.0-linux-amd64.tar.gz
  4. Execute tar xzf swarm-0.27.0-linux-amd64.tar.gz
  5. Execute sudo cp ./swarm /usr/local/bin/
  6. Check your install using the swarm info command

Whenever you need the swarm CLI, execute the first two steps of this procedure and you should be ready to go. To update the CLI, follow the same routine as for the first-time install.

Using an HTTP proxy

The swarm client you install is programmed in Go. That way we can leverage several build in features. For example you can configure a proxy for the connection between the CLI and the swarm cluster by simply setting the HTTP_PROXY env variable.

In Bash this would be:

export HTTP_PROXY="http://proxyIp:proxyPort"
