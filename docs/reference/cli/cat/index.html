swarm cat: View service definition

The swarm cat command fetches the JSON formated service definition. The output can be used to:

  • Debug a variable replacement used in the original definition.
  • Share a definition with someone else in an organization.
  • Recover a lost service definition.

Command Syntax

The swarm cat command is called using the service name to view the JSON formatted definition:

$ swarm cat <service_name>

Note: The <service_name> argument is optional when there is a proper swarm.json file in the current directory which contains the required name key.

Example with Response

$ swarm cat swacker
{
  "name": "swacker",
  "components": {
    "flask": {
      "image": "registry.giantswarm.io/bant/swacker:latest",
      "port": 5000,
      "domain": "swacker-bant.gigantic.io"
    }
  }
}

Recover to a File

The swarm cat command can be output to a file using the > redirect for STDOUT:

Example with Response

$ swarm cat swacker > swarm.json
$ cat swarm.json
{
  "name": "swacker",
  "components": {
    "flask": {
      "image": "registry.giantswarm.io/bant/swacker:latest",
      "port": 5000,
      "domain": "swacker-bant.gigantic.io"
    }
  }
}
