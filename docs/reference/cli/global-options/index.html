Global Options and Environment Variables

The behavior of the CLI can be configured and fine-tuned using a number of global command line flags and environment variables.

General syntax

Global command line options can be placed anywhere in the command line. Example:

$ swarm --debug ls -a

Environment variables are passed in the usual ways. They can be set either specificly for only one command call, by prepending the assignment to the command line. Example:

$ SWARM_ENDPOINT_URL=https://api.example.com swarm info

Alternatively, environment variables can of course be set for entire shell sessions.

Available options

-v, --verbose

This switch allows to enable or suppress detailed output.

--debug

Enable debugging output. This usually contains API calls.

-h, --help

Show help for command.

--api-endpoint

Explicitly sets the API endpoint for the command call. If used, this overrides the SWARM_ENDPOINT_URL environment variable, but only for this very command.

Example:

$ swarm --api-endpoint="https://api.example.com" info

--env

Explicitly sets the environment used. If set, this overrides the current default environment, but only for this very command.

Example:

$ swarm --env=myorganization/production ls

--config-dir

Explicitly sets the configuration directory of your swarm CLI. If set, this overrides the default directory (´$HOME/.swarm´), but only for this very command. You can use this to easily switch between users on the same computer.

swarm --config-dir=./myuser login myuser

Environment Variables

Disabling Command Line Tracking

Normally the CLI sends an HTTP header with every API request, containing the verbatim command line used to execute the CLI. Sending this header can be suppressed by setting the environment variable SWARM_DISABLE_CMDLINE_TRACKING to any non-empty string. Example:

$ SWARM_DISABLE_CMDLINE_TRACKING=true swarm create ./swarm.json --vars=password=mysecretpassword

Setting the API Endpoint

To manually set the API endpoint used, e. g. during development, use the SWARM_ENDPOINT_URL environment variable. Warning: This variable is overridden by the use of the --api-endpoint flag.

Example:

$ SWARM_ENDPOINT_URL=https://api.example.com swarm info
