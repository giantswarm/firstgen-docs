swarm status: View Service Status

The swarm status command returns information about a specific service, its components, and the instances of components.

Command Syntax

The swarm status command is called using the service name defined when the service was started.

swarm status <service_name>

The swarm status command can also be called without any arguments if a valid swarm.json file containing a service name is present in the current working directory.

Note: Use swarm ls to get a listing of all services in the current environment.

Getting status of a specific component (and its children)

The swarm status command can also be called using a component name:

swarm status <service_name>/<component>

In this case swarm status only returns the status of instances of that component. To see the status of child components use the -r or --recursive flag:

swarm status -r <service_name>/<component>

Example with Response

Note: This example uses truncated output for improved readablity.

$ swarm status weather
Service weather is up

component  image             instanceid    created              status
flask      weather:latest    1ifd72dklx8z  2015-07-14 23:05:11  up
flask      weather:latest    wz2umflmmy3t  2015-07-15 01:06:06  up
redis      redis             ft2gnsw59oj4  2015-07-14 23:03:06  up

Response Explained

The first line of the output shows the status of the service as a summary. This status is reported as the worst status of the services’ component instances. For example, if one component instance is down, the entire service will be considered down.

The remaining lines of output show a table of component instances. The table columns show which component the instance belongs to, the ID of the instance, the date and time when the instance was created, and the instance status.

Instance Status Explained

A services’ component instances may be in one of the following states:

  • up: The instance is running.
  • starting: The instance is starting.
  • stopping: The instance is stopping.
  • down: The instance is not running.
  • failed: The instance is not running and, unfortunately, there was an error.
