swarm stop: Stopping Services or Components

The swarm stop command is used to stop a service or a specific instance of a component. These instances may be restarted at a later time using the swarm start command.

When a service is stopped with swarm stop, all component instances are stopped.

The command can be called to stop an entire service without any arguments if a valid swarm.json file is present in the current working directory which represents a running service with the same name, in the current organization and environment as reported by swarm env:

swarm stop

The CLI will normally block and wait for the service to stop before it returns control to the shell. The command can also be issued in non-blocking mode using the -d or --detach flags:

swarm stop --detach

The command can also be passed a service name:

swarm stop <service_name>

Example with Response

$ swarm stop helloworld
Stopping service helloworld...
\ <--- spinner
Service helloworld is down

Warning: When a component instance is stopped, the data stored inside the instance is lost. Data persistance through stops and starts may be achieved by using volumes.

Stopping a Component

The swarm stop command may be used to stop a specific component inside a given service without stopping other components in that service. A single component is stopped by using the service name, a / separator and the component name:

swarm stop <service_name>/<component_name>

This will not stop any children of that component. To also stop all child components add the -r or --recursive flag:

swarm stop -r <service_name>/<component_name>
