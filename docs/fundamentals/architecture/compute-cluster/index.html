Compute Cluster

A Compute Cluster is the user-facing heart of the Giant Swarm platform. It comprises the machines running the User Services in form of Unit Chains. Each Compute Cluster consists of at least 3 Compute nodes, so that the user services’ components can be distributed redundantly over several machines. There can be several separate Compute Clusters per Giant OS installation. They only share the common infrastructure modules of the Control Plane.

Service Manager

The Service Manager controls the user services. It executes the creation, starting, stopping, scaling etc. of user services initiated by API calls that come in through the Control Plane.

To respond to service-specific requests for log entries or metrics (CPU and RAM usage) by end users, the Service Manager has access to cAdvisor on the local nodes (metrics) and can query the log storage on the monitoring nodes.

User Services

These are services created on request by end users. Details about user services are described in the section User Services Behind the Scenes.

Container Engine

The Docker Engine is the Container Engine used to run the containers pertaining to the user services mentioned above. It is supervised by systemd, which takes care of running, stopping, and restarting the containers with the right configuration through unit files.

Routing Layer

The routing layer takes care of routing requests from the load balancer to the containers that they are meant for and the other way around.

Service Discovery

The local Service Discovery acts as a proxy to the global service discovery and registers the services running on the compute node with the global service discovery.

Agents

On each compute node there are agents running for fleet, storage, logging, monitoring, and metrics that are interacting with the corresponding services on the Control Plane and Monitoring Cluster.
