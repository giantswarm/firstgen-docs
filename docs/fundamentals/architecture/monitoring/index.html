Monitoring Nodes

The monitoring nodes provide logging, monitoring, and alerting tools for the Giant Swarm platform.

Log Collector

Logs are forwarded from the compute nodes to the log collector, which is provided by Logstash. Logstash stores log entries as structured data to the Log Storage.

Log Storage

The Log Storage is implemented in the form of ElasticSearch. It receives log entries from the log collector. The service also responds to requests by end users to obtain log entries, which are forwarded via the API Gateway and the Service Management.

For administrative purposes, a web user interface (Kibana) is provided, allowing flexible querying of all user and system log entries.

Metrics Storage

Metrics like CPU and RAM usage from user services as well as from infrastructure modules are sent from metrics agents (cAdvisor) and collected in the metrics storage, implemented using InfluxDB. The service also responds to requests by end users to obtain metrics on their services, which are forwarded via the API Gateway and the Service Management.

For administrative purposes, a web user interface (Grafana) is provided.

Alerting

Monitoring agents on the compute nodes observe system events and trigger the alerting services when necessary. In addition, the log stream is constantly monitored for specific patterns. The external products used here are Sensu and Riemann.

For administrative purposes, an Uchiwa web user interface is provided.
