The Annotated Hello World Example

This should be your first step with Giant Swarm, a quick check that everything is working just fine.

If you already followed the Quick Start on the documentation home page without any trouble, you can skip this guide in favor of creating your first real service, unless you want some more explanation of what you just accomplished.

What you need

In order to run services on Giant Swarm, you need a Giant Swarm account. If you don’t have one yet, request an invite first. Please note that we currently have a waiting list.

Besides that, you should be on a Linux (64 bit) or Mac OS machine and be a little bit comfortable with using the command line.

Installing swarm

The Giant Swarm command line interface (CLI) is called swarm. For Mac users it’s recommended to install it via Homebrew using

$ brew tap giantswarm/giantswarm
$ brew install swarm-client

Linux users take the manual approach and download and unpack manually and move the binary to a place in their PATH, for example /usr/local/bin.

$ curl -O https://downloads.giantswarm.io/swarm/clients/0.27.0/swarm-0.27.0-linux-amd64.tar.gz
$ tar xzf swarm-0.27.0-linux-amd64.tar.gz
$ sudo cp swarm /usr/local/bin/

Check our installation reference page for details.

Logging in

Before you can do anything useful with swarm, you have to log in once. This is done by calling the swarm login command with your username as argument:

$ swarm login yourusername

You will be prompted to enter your Giant Swarm password. Note that the password won’t be displayed while you type.

Getting the code

We provide the source for our Hello World example on GitHub. To checkout the source via git, do

$ git clone https://github.com/giantswarm/helloworld.git
$ cd helloworld

Alternatively you can download a ZIP file, so you don’t even need git installed.

When looking at the contents, you’ll find there is a file called swarm.json, which is really all we need for now.

Checking the service definition

In case you want to understand what your service is doing, let’s have a look at the swarm.json file.

{
  "name": "helloworld",
  "components": {
    "webserver": {
      "image": "giantswarm/helloworld",
      "ports": 8080,
      "domains": {
        "8080": "$domain"
      }
    }
  }
}

The file defines a service called helloworld with a single component, which is called webserver. This component uses a Docker image from the public Docker registry. The image is a simple HTTP server written in Go, delivering a simple HTML page. If you are interested in the implementation details, have a look at the source code in our GitHub repository. The ports definition exposes port 8080 of the component, which is finally exposed via the domains entry via standard HTTP port 80 (Note: Access to the component from the outside is only possible via port 80 currently. Port 8080 in the domains definition refers to the exposed port of the component).

There is a nifty thing: We use a variable $domain in this file, so that you can set a domain name via the command line, so there is no need to edit that swarm.json file.

Deploying the service

We create and start the service in one step, using the swarm up command. See how we now set the $domain variable with a custom domain name using your swarm username.

$ swarm up --var=domain=helloworld-$(swarm user).gigantic.io

Now your service is created on Giant Swarm. The image you are using in your component is loaded from the Docker registry and started as a container on a machine on our cluster. Access to the service is done via the domain used in the definition.

Once everything is done, you get a success notice on the terminal.

Creating 'helloworld' in the 'yourusername/dev' environment...
Service created successfully!
Starting service helloworld...
Service helloworld is up.
You can see all components using this command:

    swarm status helloworld

The service should be running, so let’s check it in a browser. Note that the URL contains your Giant Swarm username and looks like helloworld-yourusername.gigantic.io. If everything went fine, you’ll see a blue page with some pleasantly positive content.

Cool. Take a breath and enjoy your achievement, before it’s time to clean up.

The following two commands will stop and delete the service to free resources on the cluster.

$ swarm stop helloworld
$ swarm delete helloworld

Note that the delete command requires confirmation to actually delete the service.

Congratulations! You are now prepared for greater achievements.
